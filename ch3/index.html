2.5 Protocols

A protocol is a set of rules for communication

{ }

Syntax

Defines the format and structure of data, such as packet fields, headers, and payload.

🔎

Semantics

Defines the meaning of each field, such as IP address, port number, flags, or error code.

⏱️

Timing

Defines when to send, how long to wait, timeout, ACK behavior, and flow control.

Protocols are organized as a stack, from application protocols down to physical transmission technologies.

Check understanding

Common mistakes to avoid

1. Application Layer ≠ the app itself

It means network services and protocols that applications use, such as HTTP, DNS, SMTP, and FTP.

2. TCP/IP Application is broad

It covers roles similar to OSI Application, Presentation, and Session layers.

3. MAC address is link-by-link

It can change at each link. Switches use MAC addresses inside a LAN.

4. IP address is end-to-end routing identity

Routers use IP addresses to forward packets between networks.
